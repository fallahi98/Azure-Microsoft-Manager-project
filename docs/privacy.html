Policy

Privacy Policy

Effective date: June 17, 2026

Overview

Azure Client Manager is a private administrative project for managing client records, case notes, reminders, email notifications, and SMS notifications. This policy explains the types of information the project is designed to process and how that information should be handled.

Information Processed

The application may process client names, addresses, ZIP codes, cities, email addresses, phone numbers, case details, reminder notes, and communication history entered by an authorized user.

The public GitHub Pages site does not collect account logins, payment information, or form submissions.

How Information Is Used

  • To store and organize client records.
  • To manage case status and case history.
  • To schedule reminders and administrative notes.
  • To send configured email or SMS notifications when requested by the application workflow.

Storage and Security

Operational data is intended to be stored in the configured database and communication services controlled by the project administrator. Credentials should be stored only in local, ignored configuration files or secure environment variables, not in public source code.

Sharing

Information should not be sold or shared for advertising. Information may be processed by configured infrastructure providers, such as database, email, SMS, hosting, or repository services, only as needed to operate the project.

Retention

Client and case data should be retained only as long as needed for the administrative purpose for which it was entered, unless a longer retention period is required by business, legal, or compliance needs.

Contact

Questions about this policy or the project should be directed to the project administrator.
