SMS Compliance

Effective date: June 21, 2026

Program Description

Client Manager is a private administrative client-management project. SMS functionality is used only for controlled client reminder and testing workflows managed by the project administrator. The current SMS workflow uses configured SMTP email settings and a carrier email-to-SMS gateway, such as Verizon vtext.com, instead of Azure Communication Services.

SMS Delivery Method

Messages are sent from the administrator's configured email account to the recipient carrier gateway address. For Verizon numbers, the application formats the phone number as a vtext.com gateway address. Delivery depends on the recipient carrier supporting email-to-SMS gateway delivery.

Message Recipients

Phone numbers are manually entered by an authorized administrator. The application is not used for public marketing, advertising, mass messaging, lead generation, or purchased contact lists.

Consent and Opt-In

Messages are sent only when the administrator has permission to contact the recipient or when the administrator is testing with their own phone number. No third-party users are automatically enrolled through this public website.

Message Purpose

  • Administrative reminders related to client records or case notes.
  • Application testing by the project administrator.
  • Operational notifications configured inside the private application.
  • Email-to-SMS delivery through the configured carrier gateway.

Opt-Out

If a recipient asks not to receive messages, the administrator should remove that phone number from future SMS workflows. Any production deployment should honor STOP, unsubscribe, or equivalent opt-out requests where required. Because email-to-SMS gateways may not automatically process carrier opt-out keywords, the administrator is responsible for maintaining the opt-out list.
